# Alembic Migrations Quick Reference

A quick reference guide for common Alembic migration commands. For detailed documentation, see [DATABASE_MIGRATIONS.md](../DATABASE_MIGRATIONS.md) in the project root.

## Prerequisites

```bash
# Ensure you're in the backend directory
cd backend

# Verify Alembic is installed
pip show alembic
```

## Common Commands

### Check Status

```bash
# Show current migration version
alembic current

# Show migration history
alembic history

# Show detailed history with full descriptions
alembic history --verbose
```

### Apply Migrations (Upgrade)

```bash
# Apply all pending migrations
alembic upgrade head

# Upgrade by one version
alembic upgrade +1

# Upgrade to specific revision
alembic upgrade 001
```

### Rollback Migrations (Downgrade)

```bash
# Rollback one version
alembic downgrade -1

# Rollback to specific revision
alembic downgrade 001

# Rollback all migrations
alembic downgrade base
```

### Create New Migrations

```bash
# Auto-generate migration from model changes
alembic revision --autogenerate -m "Add user preferences table"

# Create empty migration template
alembic revision -m "Custom data migration"
```

### Preview SQL

```bash
# Show SQL for upgrade without executing
alembic upgrade head --sql

# Show SQL for downgrade without executing
alembic downgrade -1 --sql
```

## Initial Setup Workflow

```bash
# 1. Ensure database exists
createdb chatbot_db -U chatbot

# 2. Apply all migrations
alembic upgrade head

# 3. Verify tables created
psql -U chatbot -d chatbot_db -c "\dt"
```

## Development Workflow

### Adding a New Feature

```bash
# 1. Update models in app/models/database.py
# 2. Generate migration
alembic revision --autogenerate -m "Add feature X"

# 3. Review the generated migration file
cat alembic/versions/*_add_feature_x.py

# 4. Apply migration
alembic upgrade head

# 5. Test your changes
pytest

# 6. Commit migration file to git
git add alembic/versions/*_add_feature_x.py
git commit -m "Add database migration for feature X"
```

### Testing Migration Rollback

```bash
# 1. Apply migration
alembic upgrade head

# 2. Test rollback
alembic downgrade -1

# 3. Re-apply migration
alembic upgrade head

# 4. Run tests to verify
pytest
```

## Troubleshooting

### "Can't locate revision"

```bash
# Check current state
alembic current

# Check available migrations
alembic history

# Force stamp if needed (use with caution!)
alembic stamp head
```

### "Target database is not up to date"

```bash
# Simply upgrade to latest
alembic upgrade head
```

### Models not detected by autogenerate

Ensure all models are imported in `alembic/env.py`:

```python
from app.models.database import (
    User, Document, ChatSession, ChatMessage, Author, Post
)
```

## Configuration Files

- **alembic.ini**: Main configuration
- **alembic/env.py**: Environment setup (imports models)
- **alembic/script.py.mako**: Template for new migrations
- **alembic/versions/**: Migration files

## Database Connection

Alembic automatically uses `settings.DATABASE_URL` from `app/utils/config.py`.

Default: `postgresql://chatbot:password@localhost:5432/chatbot_db`

Override in `.env`:
```env
DATABASE_URL=postgresql://user:pass@host:port/dbname
```

## Best Practices

1. **Always review autogenerated migrations** before applying
2. **Test migrations locally** before deploying
3. **Keep migrations small and focused** on single features
4. **Never modify applied migrations** - create new ones instead
5. **Backup production database** before major migrations
6. **Use descriptive migration messages**

## Quick Example

```bash
# Add new column to User model
# Edit app/models/database.py:
#   timezone = Column(String(50), default='UTC')

# Generate migration
alembic revision --autogenerate -m "Add timezone to users"

# Review
cat alembic/versions/*_add_timezone_to_users.py

# Apply
alembic upgrade head

# Verify
alembic current
```
